webpack 性能优化

在大型项目中打包一次要花费非常多的时间，因此就是使用一些方法来提升打包效率

跟上技术的迭代（Node、Npm、Yarn）

升级 webpack 的版本，或者升级 Node、Npm、Yarn 版本，因为每一个版本做更新内部要做很多优化

在尽可能少的模块上应用 Loader

可以通过 exclude 或者 include 约定只有某一些文件夹下的文件使用 loader，让 loader 的转化和编译的过程减少，因此可以提升 webpack 打包速度，合理使用 exclude 和 include 能够降低 loader 的使用频率，从而提高打包速度

Plugin 尽可能精简并确保可靠

比如在开发环境下不需要做代码压缩，而在生产环境下需要做代码压缩，因此在开发阶段节约了打包时间，让打包速度更快，并且建议使用 webpack 官方网站推荐的插件，因为这些插件性能往往经过了官方的测试，比较快的，而在一些个人的插件时，尽管会帮助我们解决一些问题，但是其插件的性能得不到保证，有可能降低打包速度

控制包文件大小

在做打包的时候，应该让打包的文件尽可能的小

有的时候在页面里面引入一些没有用的模块，并且没有使用，如果还没有配置 tree shaking，这就会导致代码里面有很多冗余的代码，这些冗余的代码就会拖累 webpack 的打包速度，因此很有必要配置 tree shaking

thread-loader、parallel-webpack、happypack 多进程打包

webpack 默认是通过 node.js 来运行的，所以是一个单进程的打包过程，因此可以借助 node 里面的多进程帮助提升打包速度

合理使用 sourceMap

在打包时生成 sourceMap，其越详细，打包速度越慢，因此要在不同环境下，使用最优的 sourceMap 生成代码调错文件，一方面帮我们挑问题，一方面提升打包速度

结合 stats 分析打包结果

哪一个模块打包时间长，哪个时间短都可以分析出来，然后对其进行优化，因此也是在实战中常用的手段

开发环境内存编译

在开发时，用 devServer 做打包时不会生成 dist 目录，会把编译的文件放到内存中，内存的读取肯定比硬盘的读取速度要快很多，所以采取这种手段也可以让 webpack 性能提升很多

开发环境无用插件剔除

比如开发环境并不需要代码压缩，因此要将 mode 设置为 development

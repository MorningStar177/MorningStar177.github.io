Bootstrap

  • 一个 CSS 框架
  • twitter 出品
  • 提供通用基础样式

https://getbootstrap.com/

  • 兼容 IE 10+
  • 使用 flexbox 布局
  • 抛弃 Normalize.css
  • 提供布局和 reboot 版本

Bootstrap 提供

  • 基础样式
  • 常用组件
  • JS 插件

使用方法

不同方式引入

简单的表单，无任何样式，不输入密码会文字提示

引入 bootstrap

引入后刷新页面，样式会初始化，比如 body 的 margin = 0

grid 下面会有表格的样式说明，例如 col-6，代表网格占六份，offset-3，页面共9，右移3，这样 container 会居中

表单样式 ，例如水平表单 ，需要在每个外部 div 上添加 row mb-3 的类名，内部的每个 label 需要加入类名 col-sm-2 col-form-label，并且需要将 input 输入框用 class 为 col-sm-10 的 div 包裹起来，输入框也需要加入类名

按钮样式 ，比如选择 primary 按钮，则加入类名 btn btn-primary

Bootstrap JS 组件

用于组件交互

  • dropdown（下拉）
  • modal（弹窗）
  • …

Bootstrap JS 组件基于 jQuery 写的，因此需要引入 jQuery，同时还依赖第三方库 Popper.js（单独用来处理弹出框的 JS 插件），最后引入 bootstrap.js

依赖

  • jQuery
  • Popper.js
  • bootstrap.js

使用方式

  • 基于data- 属性（不太常见）
  • 基于 JS API

实例

通过按钮中的 data-bs-target 属性与指定显示对话框的 id 匹配

通过关闭按钮中的 data-bs-dismiss 属性与 modal 展示状态连接实现切换

大部分情况下不会使用 bootstrap JS 组件

Bootstrap 定制化

对 Bootstrap 风格进行修改，以适应项目的风格

  • 使用 CSS 同名类覆盖
  • 修改源码重新构建（直接修改 scss 源文件，重新编译）
  • 引用 scss 源文件修改变量（把 Bootstrap 当做 mixin 使用）

CSS 同名类覆盖

场景简单可以用，复杂不推荐使用

好处：操作简单

坏处：没有办法再去使用 Bootstrap 写好的变量之间的关系，并且要非常全面的覆盖 class，否则会有 bug

demo

按钮与输入框的 focus 都是蓝色并且都是圆角

修改这两个样式

按钮

  • 调试工具定位到按钮的类名（类中有背景色定义）（.btn-primary）
  • 找到按钮中具有圆角属性的类名（.btn）
  • 直接修改

输入框

操作于修改按钮类似，圆角类名（.form-control）、focus 类名（.form-control:focus）

找 focus 的方法：打开调试，找到 input 标签，选中标签，鼠标右键选中 Force state > :focus

修改完毕

CSS 面试真题

Bootstrap 的优缺点

  • 优点：预处理器代码（scss、less）结构合理，网格系统、响应式设计思想值得借鉴，提供现成的样式可以直接用
  • 缺点：定制较为繁琐（类名覆盖、修改源码重新构建），体积大（样式比较多）

Bootstrap 如何实现响应式布局

  • 原理：通过 media query 设置不同分辨率的 class
  • 使用：为不同分辨率选择不同的网格 class

如何基于 Bootstrap 定制自己的样式

  • 使用 CSS 同名类覆盖（上手简单，但容易遗失出 bug）
  • 修改源码重新构建（合理，无隐患，但需要了解 bootstrap 结构）
  • 引用 scss 源文件 + 修改变量（当做类库使用，挑选模块，并且对模块定制，按需使用，控制体积，但需要对其结构关系了解的更加透彻）

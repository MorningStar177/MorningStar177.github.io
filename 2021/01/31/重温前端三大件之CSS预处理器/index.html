CSS 预处理器

  • 基于 CSS 的另一种语言
  • 通过工具编译成 CSS
  • 添加了很多 CSS 不具备的特性（比如变量）
  • 能提升 CSS 文件的组织

less、sass（scss）

CSS 预处理器提供的功能

  • 嵌套 - 反映层级和约束
  • 变量和计算 - 减少重复代码
  • Extend 和 Mixin 代码片段（反复不断使用）
  • 循环 - 适用于复杂有规律的样式
  • import - CSS 文件模块化

less import

预处理器处理模块化

css import 的写法问题：不会做任何合并，在加载时候也不会复用任何连接，只能依次加载，这样会增加 http 请求书，造成性能上的问题

因此预处理器做了改进，会把不同的 import 整合编译到一起

预处理器的变量可以跨文件使用，只要正确的 import

CSS 预处理器框架

按需使用其他人写好的代码，无需整篇代码复用

  • SASS - Compass
  • Less - Lesshat / EST
  • 提供现成的 mixin
  • 类似 JS 类库 封装常用功能

CSS 面试真题

常见的 CSS 预处理器

  • Less（Ndoe.js 写的）
  • Sass（用 Ruby 写的，有Node版本）

预处理器的作用

  • 帮助更好地组织 CSS 代码（通过变量、mixin 等）
  • 提高代码复用率
  • 提升可维护性

预处理器的能力

  • 嵌套 - 反映层级和约束
  • 变量和计算 - 减少重复代码
  • Extend 和 Mixin 代码片段（反复不断使用）
  • 循环 - 适用于复杂有规律的样式
  • import - CSS 文件模块化（无需考虑加载等问题，强行写到一个 CSS 文件中）

预处理器的优缺点

  • 优点：提高代码复用率和可维护性
  • 缺点：需要引入编译过程、有学习成本（独立语法）

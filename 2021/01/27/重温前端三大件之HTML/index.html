HTML 重要属性

面试题：如果是 Ajax 请求，并不通过 Form 的 subbmit 提交，是否需要 Form 元素？

（技术上并不一定需要，但还是建议添加 Form 元素）

第一：因为有 subbmit 和 reset 的特性在，可以利用，如果没有 Form 元素则不能使用

第二：当有 Form 的时候，可以通过一些方式去批量的获取表单数据

第三：当有 Form 的时候，可以跟一些验证组件结合做表单验证，如果没有 Form 用起来不方便

第四：当有 Form 的时候，对用户体验会有提升，比如浏览器会记住用户名密码等

因此只要有表单的地方都建议使用 Form

如何理解 HTML

  • HTML “文档”
  • 描述文档的“结构”
  • 有区块和大纲

工具（W3C 的大纲算法）：https://h5o.github.io/

HTML5 新增內容

新区块标签

  • section
  • article
  • nav
  • aside

表单增强

  • 日期、时间、搜索
  • 表单验证
  • Placeholder、autofocus

HTML5 新增语义

  • header / footer 头尾
  • section / article 区域
  • nav 导航
  • aside 不重要内容(广告、友情链接等)
  • em / strong 强调
  • i icon 做图标

实现一个页面语义化，要先对页面功能作区分，然后依次填入对应语义标签

问：HTML5 这么大变动只有这么一点东西吗？

答：既是也不是，考虑 HTML 作为一个静态页面或作为一个文档，新增的东西不是太多。但因为 HTML5 新增的很多是属于 WebAPI 的内容，JS 调用的能力，例如本地存储、定位等，在 HTML5 中并没有很直观的反馈

HTML 元素分类

按默认样式分

  • 块级 block（默认占据整行不会给其他元素留空间）
  • 行内 inline（也叫内联元素，无规则形状，可跟其他元素挤在一起）
  • inline-block（对外表现像 inline，与其他元素和谐挤在一起；对内表现像 block 元素，有自己的尺寸，比如宽高等）

按内容分

了解即可

https://html.spec.whatwg.org/multipage/dom.html#phrasing-content

HTML 元素嵌套关系

  • 块级元素可以包含行内元素
  • 块级元素不一定能包含块级元素（比如 p 内 div 不可以）
  • “行内元素一般不能包含块级元素”（span 里包 div 不可以，a 元素可以包 div）

为什么 a>dⅳ 是合法的

在 HTML 4.01 中是不合法的，但是很多场景下，广告最外层包含着 a 标签，因此 HTML5 中做了合法化的分类

同时可以查看 content-model，看嵌套的类型是否属于合法类型

HTML 元素默认样式

  • 默认样式的意义
  • 默认样式带来的问题
  • CSS Reset

比如 body 默认 margin、ul 元素上下 margin 与 padding

或者 CSS Reset 简介但有性能争议的写法，仅需四行

HTML 面试真题

doctype 的意义是什么

  • 让浏览器以标准模式渲染
  • 让浏览器知道元素的合法性

HTML、XHTML、HTML5 的关系

  • HTML 属于 SGML（SGML 是非常通用的标记语言）
  • XHTML 属于 XML，是 HTML 进行 XML 严格化的结果
  • HTML5 不属于 SGML 或 XML，比 XHTML 宽松

HTML5 有什么变化

  • 新的语义化元素
  • 表单增强
  • 新的API（离线、音视频、图形、实时通信、本地存储、设备能力）
  • 分类和嵌套变更

em 和 i 有什么区别

  • em 是语义化的标签，表强调
  • i 是纯样式的标签，表斜体
  • HTML5 中 i 不推荐使用，一般用作图标

语义化的意义是什么

  • 开发者容易理解
  • 机器容易理解结构（搜索、读屏软件）
  • 有助于 SEO

哪些元素可以自闭合

  • 表单元素 input
  • 图片img
  • br hr
  • meta link

HTML和DOM的关系

  • HTML 是“死”的
  • DOM 由 HTML 解析而来，是活的
  • JS 可以维护 DOM

property 和 attribute 的区别

  • attribute 是“死”的
  • property 是“活”的
  • 改变时互不影响

form 的作用有哪些

  • 直接提交表单
  • 使用 submit / reset 按钮
  • 便于浏览器保存表单
  • 第三方库可以整体提取值
  • 第三方库可以进行表单验证

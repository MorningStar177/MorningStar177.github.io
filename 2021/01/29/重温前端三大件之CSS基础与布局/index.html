    行高是由 linebox 决定的，linebox 是由多个 inlinebox 组成的，inlinebox 的高度会决定行高的高度

  • 行高相关的现象和方案

    图片下面的空隙如何去除？ （非常重要的面试题）

    分析：原因是 img 默认是 inline 的元素，要遵守行高的构成，因此会按照 baseline 对齐，这就意味着在 baseline 与 底线之间会有一段距离，这就是空隙产生的原因

    解决：第一种方式，为图片添加 vertical-align: bottom ；第二种方式，直接 display: block

    转 base64 工具

    好处：减少 http 请求数

    缺点：

    1.文件体积增大，图片本身体积会变为原来的 4/3，并且原来图片是单独文件，转换后相当于放在了 css 文件中，这会导致 CSS 文件本身体积变大，因此一般只适用于非常小的图标

    2.base64 在传输上增大了体积的开销，增大了解码的开销

    3.难以维护，因为没有图，因此一般会在构建打包的时候才转成 base64

CSS Hack

面试中考的几率很小，但是如果问到怎么样处理浏览器兼容性问题的时候，CSS Hack 可以作为一种解决方式，虽然并不是最好且最优雅的

看起来不一定合法或有一些合法但是非常偏门的用法，这些用法的结果是只在一些特定的浏览器中生效，并非全部浏览器生效，在一部分浏览器中生效的写法就叫做 CSS Hack

作用：比如写了一个属性 width: 100px，但是在 IE 中出来是 120 px，因此就需要写成 width: 80px，这就是一种处理浏览器兼容性的写法

  • Hack 即不合法但生效的写法
  • 主要用于区分不同浏览器
  • 缺点：难理解、难维护、易失效
  • 替代方案：特性检测
  • 替代方案：针对性加 class（比如加 IE6 的 class）
  • 简单粗暴的解决方案，能立刻生效

写 hack 的时候注意，标准属性写到前面，hack 写到后面

CSS 布局

CSS 知识体系的重中之重

  • 早期以 table 为主（简单）
  • 后来以技巧性布局为主（难）
  • 现在有 flexbox/grid（偏简单）
  • 响应式布局是必备知识

常用布局方法

  • table 表格布局
  • float 浮动+ margin
  • inline-block 布局
  • flexbox 布局

table 表格布局

以前用的多，现在基本不会用，如果只会表格布局是减分项

不用 table 标签方式利用 table 方式布局

布局属性

盒模型

display / position

display 确定元素的显示类型

  • block / inline / inline-block

position 确定元素的位置

  • static（默认的文档流）

  • relative （相对于元素本身，特点为偏移的时候不会改变占据的空间）

  • absolute (相对于上级中最近一级带有定位如 absolute / relative 的元素，一直找到 body 为止，脱离文档流，相当于独立存在，自身不再占有位子，不会对其他元素造成影响)

  • fixed（脱离文档流不会对其他元素布局造成影响，相对于可视区域固定的）

层级（需要有 absolute、relative、fixed 定位的元素）：z-index，数字高就覆盖另一元素

flexbox

目前的布局方式，但并没有大规模被使用，因为有兼容性问题（低版本 IE 完全不支持 flex），但在移动端，例如微信小程序在用

  • 弹性盒子
  • 盒子本来就是并列的
  • 指定宽度即可

flex: 1，代表平分宽度后占一份，如果仅有一个元素则类似于 width: 100%

如果想某个元素固定宽度，则在该元素身上添加如下属性

其他元素平分盒子剩下宽度

float

特点

  • 元素“浮动”
  • 脱离文档流（不会对其他元素造成干扰）
  • 但不脱离文本流

对自身的影响

  • 形成 “块”（BFC）

    如对 span 这种 inline 元素设置 float 之后，可以设置宽高

  • 位置尽量靠上

  • 位置尽量靠左（右）

对兄弟的影响

  • 上面贴非 float 元素
  • 旁边贴 float 元素
  • 不影响其它块级元素位置
  • 影响其它块级元素内部文本

对父级元素的影响

  • 从布局上“消失”

  • 高度塌陷

    解决办法

    第一种：对其父元素加 bfc，接管宽高

    第二种：利用元素（伪元素）撑起父级元素

面试题：用 float 实现 两栏或者三栏布局

两栏

inline-block 布局

  • 像文本一样排 block 元素
  • 没有清除浮动等问题
  • 需要处理间隙

inline-block 类似于文字间的缝隙，解决办法：

第一种：将父元素字体设置为 0，然后再分别设置子元素字体

第二种：直接简单粗暴把两个 div 之间设置为并列，或者注释方式

推荐第一种

inline-block 适合定宽的布局

响应式设计和布局

移动端时代，响应式布局是必须掌握的内容，可以帮助网页更好的适配不同大小的移动端、pc 端

  • 在不同设备上正常使用
  • 一般主要处理屏幕大小问题

主要方法

  • 隐藏（侧边栏，个人信息等通过点击显示） + 折行（特性列表） + 自适应空间（比如 logo 固定宽度，其右边自适应）
  • rem / viewport / media query

问：怎样在一个页面适配移动端

答：加 viewport

通过隐藏方式处理内容

通过 rem

rem 单位并不精确（小数），对精确度要求十分高的布局不能使用

CSS 面试真题

实现两栏（三栏）布局的方法

  • 表格布局
  • float + margin 布局（主流，兼容性好）
  • inline-block 布局（注意间隙的处理）
  • flexbox 布局（现代化布局方式，但是兼容性不好）

position:absolute / fixed 有什么区别

  • 前者相对最近的 absolute / relative
  • 后者相对屏幕（viewport）

display:inline-block 的间隙

  • 原因：空白字符间距
  • 解决：消灭字符或者消灭间距

如何清除浮动

为什么要清除浮动：浮动的元素不会占据父元素的布局空间，也就是父元素布局时不会管浮动元素，有可能会超出复原素，从而影响其他元素，所以需要清除浮动

清除方法：

  • overflow:hidden（auto）
  • ::after { clear:both }

如何适配移动端页面

  • 宽度尺寸的适配：加 viewport，也就是页面宽度要和移动端宽度适配
  • 做一些大小方面的适配：rem / viewport / media query
  • 设计上：隐藏（不重要的不显示）、折行、自适应（留下自适应空间，可大可小）
